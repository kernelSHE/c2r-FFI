# C2RS 仓库规则（硬约束）

## 1. 禁止 `extern "C"`

- **全仓库禁止**在生成的或手写的 Rust 代码中出现 `extern "C"`。
- 代码审查与 CI 必须检查此项；违规即失败。

## 2. 转换管线

- 管线**必须**为：**C → IR（可序列化 JSON）→ Rust**。
- Rust 代码**只**从 IR 生成，不得从 C 源码直接生成 Rust。

## 3. 结构保持

- **N 个 `.c` ⇒ N 个同名 `.rs`**，路径尽量一致。
- 函数一一对应，通过 `mapping.json` 可追踪 C 符号 ↔ Rust 符号。

## 4. Unsafe 策略

- 目标：尽量 **0 unsafe**。
- 若阶段性无法完全做到，必须：**隔离、最小化、可审计**（集中到少数模块并文档化）。

## 5. LLM 角色

- LLM **只能**用于「修复回路」（如修复编译错误、风格），**不能**替代 **IR → Rust 的确定性生成**。
